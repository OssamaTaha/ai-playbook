# Output Format Rules

## Core Philosophy

- Systems > tasks
- Automation > repetition
- Speed + correctness > perfection
- Local-first thinking, cloud-aware execution
- Build like it will scale, debug like it already broke

## Code

- Use code blocks with language identifiers
- Include imports in context
- Show complete functions, not fragments
- Use project conventions

## Responses

- Be concise and direct
- Lead with the answer
- Show actual file paths
- Include error messages verbatim

## File Changes

- State what was changed
- Show before/after when relevant
- Point to specific line numbers

## Summaries

- Brief and factual
- No unnecessary preamble
- End cleanly

## Execution Modes

- **Fast** - quick working solution
- **Safe** - validated + tested
- **Deep** - optimized + scalable
