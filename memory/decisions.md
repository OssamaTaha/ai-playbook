# Decisions

## Documented Choices

Record why decisions were made, not just what.

### Format

```
## [Decision Title]

**Context**: What was the situation?
**Decision**: What was chosen?
**Rationale**: Why this choice?
**Alternatives**: What was considered?
**Consequences**: Impact of decision
```

### Categories

- Architecture choices
- Technology selections
- Tradeoffs made
- Priority decisions
- Performance vs maintainability

## Examples

- Chose PostgreSQL over MongoDB for relational data
- Implemented caching layer for performance
- Selected async processing for scalability
- Used event sourcing for audit trail
